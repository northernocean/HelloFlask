  • /earthquakes/p1 Postgres

    Route with a postgres database data source.

  • /earthquakes/p2 Postgres / Api

    Route with a postgres database datasource, but routed via an api route served from the same project. Mostly to show it works, since if you have the database already accessible in the same project you do not really an api to access the data. Perhaps this is nice if you intend to later move the database out of the project.

  • /earthquakes/s1 Sqlite

    Route with a local sqlite (readonly) database. Per Heroku docs, sqlite is not recommended. Probably it is fine for read-only access. This is a good option to get experience with a database data source but also keeping the project simple with a local sqlite database file that you can use exactly the same in development locally as well as in the deployed application at heroku.
